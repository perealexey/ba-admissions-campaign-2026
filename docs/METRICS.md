# Метрики бакалаврской приёмной кампании (M1–M9) — справочник

Формализация слоя метрик по итогам разведки данных (`docs/FINDINGS.md`,
`docs/HANDOFF_FOR_METRICS_DESIGN.md`) — все методологические развилки из
handoff-документа решены (пользователем или выведены из данных), здесь только
итоговые формулы, константы и обработка краёв, готовые к реализации в
`scripts/campaign_metrics_bak.py` по образцу `summer_campaign_26/scripts/campaign_metrics.py`.
Не переоткрывать решённые вопросы без явного запроса — см. `HANDOFF_FOR_METRICS_DESIGN.md`
для истории и обоснования каждого решения.

## 0. Входные данные и допущения

- `bak_data/raw/*.json` — обычные конкурсные группы (Бюджетные места,
  Отдельная квота, Особое право, С оплатой обучения), одна запись = один
  `competitiveGroupId`.
- `bak_data/raw_target_quota/*.json` — 32 уникальных пула целевой квоты,
  каждый может относиться к нескольким `competitiveGroupId`-дублям одной
  программы (заказчики) — при агрегации по программе объединять через
  `nunique(idEpgu)`, не суммировать построчно по пулам.
- `bak_data/skipped_foreigners.csv` — 86 групп «Иностранцы — отдельный
  конкурс», в метрики не входят, это решённое исключение, не пробел.
- `docs/{moscow,spb,nnov,perm}_supply.rtf` — источник КЦП, требует парсинга
  (см. §3) в `data/supply_demand_bak.csv`.
- **Канонiчный join-ключ программы — `educationProgramId`, никогда
  отображаемое имя** (10 подтверждённых «близнецов» — одно имя и филиал,
  разные `educationProgramId`, разные `trainingDirection`, см. `FINDINGS.md`).

**Три несопоставимых пространства приоритетов** (ключевое отличие от
магистратуры, где `place_type` был бинарным с двумя независимыми шкалами):

1. **Узкая бюджетная семья** — Бюджетные места + Отдельная квота + Особое
   право. Общая нумерация приоритета внутри семьи (99.3–99.6% совпадений на
   сырых данных). Перед расчётом метрик схлопывается до одной строки на
   `(idEpgu, educationProgramId)` по правилу ниже.
2. **С оплатой обучения** — независимая нумерация приоритета (аналог
   `commercial` из магистратуры).
3. **Целевая квота** — независимая нумерация приоритета (подтверждено:
   совпадение с бюджетной семьёй всего 10.4%, это НЕ часть семьи, несмотря
   на то, что это реальные бюджетные места). Не участвует в M4/M5/M6,
   отдельная диагностическая колонка — решение пользователя, см. §1.

**Правило схлопывания узкой бюджетной семьи** (разобрано поимённо на всех 47
расходящихся случаях из 74 728 пар человек-программа, см. `HANDOFF...md` §6.1):
для `(idEpgu, educationProgramId)` с несколькими строками внутри семьи —
брать `priority` из строк со статусом, отличным от «Отозвано поступающим»;
если таких несколько и они совпадают — общее значение; если совпадений нет
(только один человек на всю выборку, `idEpgu=1785708`) — минимальный
`priority` как тай-брейк.

```python
ACTIVE_STATUSES_EXCLUDE = {"Отозвано поступающим", "Отклонено вузом"}

def collapse_budget_family(rows: pd.DataFrame) -> pd.Series:
    """rows: все строки бюджетной семьи одного (idEpgu, educationProgramId)."""
    active = rows[~rows["participantStatus"].isin(ACTIVE_STATUSES_EXCLUDE)]
    pool = active if len(active) > 0 else rows  # все отозваны — берём любую, приоритет не важен
    return pool.loc[pool["priority"].idxmin()]
```

**Константа `K_SERIOUS = 5`** — порог «серьёзной» заявки (`priority ≤
K_SERIOUS`), решение пользователя (не 2, как в магистратуре — распределение
приоритетов в бакалавриате растянутее, при пороге 2 «серьёзными» оказались бы
47.7% всех заявок против 84.6% при пороге 5, см. `HANDOFF...md` §6.3).

**Активная заявка** (для M1–M3) — `participantStatus` НЕ в `{"Отозвано
поступающим", "Отклонено вузом"}` (6.41% всех заявок на срезе 2026-07-08).
Первое приближение по смыслу названий статусов, не подтверждённая бизнес-логика
(документации от приёмной комиссии нет) — пересмотреть, если появится.

**Балл для ранжирования** — `sumCompetitiveScore` для всех треков, кроме
целевой квоты, где использовать `sumCompetitiveScoreTarget` (отдельная шкала
учёта индивидуальных достижений для целевого приёма, расхождение с обычным
баллом — 4 случая из 128 743 на срезе). Официальный ранг `indexNumberInCompList`
пуст у 100% заявок на срезе 2026-07-08 (кампания не финализирована), сверить
формулу с ним нельзя — балл рассчитан и подтверждён напрямую:
`sumCompetitiveScore = sumEntranceTestScore + achievementsSum` без исключений.

## 1. Метрики — определения

Все метрики per-`educationProgramId`. Три колонки везде, где это осмысленно
(бюджетная семья / коммерция / целевая квота) — **не сумма**, целевая квота
несопоставима по приоритету с остальными (см. §0).

### M1 — Заявления
- `n_applications_budget_family` — `nunique(idEpgu)` среди активных строк
  схлопнутой узкой бюджетной семьи.
- `n_applications_commercial` — число активных строк «С оплатой обучения»
  (дублей внутри трека нет, аналог магистратуры).
- `n_applications_target_quota` — `nunique(idEpgu)` среди активных строк по
  всем пулам целевой квоты, относящимся к программе.

### M2 — Уникальные абитуриенты
`n_unique_applicants_total` — `nunique(idEpgu)` по программе в целом, по всем
трём каналам вместе (человек может быть одновременно в бюджетной семье и на
платном — считается один раз).

### M3 — Серьёзный объём
`priority ≤ K_SERIOUS` (после схлопывания для бюджетной семьи), раздельно:
`n_serious_budget_family`, `n_serious_commercial`. **Целевая квота исключена**
(её priority не про то же самое, что «серьёзность» в остальных треках —
см. §0 и решение по M4 ниже).

### M4 — Рейтинг желанности (бюджет+квоты И платное, БЕЗ целевой квоты)
**Пересмотрено 2026-07-08.** Изначально M4 считался только на узкой
бюджетной семье, как в магистратуре — но в бакалавриате это создавало
слепую зону: программы совсем без бюджетных мест (например «Управление в
креативных индустриях» — целиком платная) молча исключались из рейтинга,
хотя реальный спрос на них сопоставим с бюджетными программами (384 общих
уникальных абитуриента с «Медиакоммуникациями» на одном срезе).

**Решение**: желанность считается ОТДЕЛЬНО и НЕЗАВИСИМО для каждого из двух
каналов (`desirability_budget` — на схлопнутой узкой бюджетной семье,
`desirability_commercial` — на «С оплатой обучения», та же формула
эмпирического Байеса, что и в магистратуре, `summer_campaign_26/docs/METRICS.md`
§2), а затем объединяется в `desirability_combined` через **inverse-variance
weighting** (взвешивание по обратной апостериорной дисперсии):

```
weight_budget = 1 / posterior_var_budget   # var Beta-Binomial модели each канала
weight_comm   = 1 / posterior_var_comm
desirability_combined = (weight_budget·desirability_budget + weight_comm·desirability_commercial)
                         / (weight_budget + weight_comm)
```

Вес — не произвольная настройка, а прямое следствие уже существующей модели:
чем больше данных и увереннее оценка в канале, тем больше его вклад. Если
канала нет вовсе (программа без бюджета или без платного) — его вес
автоматически ноль, `desirability_combined` гладко становится равен
единственному существующему каналу — не нужен отдельный case для «нет
данных».

**Почему объединение статистически оправдано, а не притянуто за уши**:
проверено эмпирически — `desirability_budget` и `desirability_commercial`,
посчитанные полностью независимо, коррелируют на **r=0.94** по всем 121
программе с данными в обоих каналах на срезе 2026-07-08 (`M4_corr_budget_commercial`
в `campaign_metrics_meta_bak.json`). Это означает, что оба канала измеряют,
по сути, одну и ту же скрытую величину с разным шумом — комбинировать их
статистически корректно (в отличие от целевой квоты, где корреляция с
бюджетом всего 10.4%, см. §0 — там объединение было бы некорректным).

**Важный интерпретационный нюанс (не баг, показывать пользователю дашборда,
не только держать в голове разработчика)**: в топе объединённого рейтинга
часто оказываются нишевые программы, которые существуют только в платном
канале, с очень высокой сырой долей приоритета №1 (например 50 из 56 = 89%).
Это не артефакт шума — проверено, `n` там от 24 до 84, выше порога
`MIN_N_FOR_VARIANCE_ESTIMATION`. Причина содержательная: у чисто платной
нишевой программы нет бесплатной альтернативы, которая привлекала бы
малозаинтересованных «на всякий случай» абитуриентов — подаются только
по-настоящему замотивированные, поэтому доля первого приоритета там
структурно выше, чем у массовых конкурентных программ с бюджетными местами.
Это законный сигнал, просто другой природы, а не то же самое, что «желаннее
для всех» — дашборд должен явно объяснять это рядом с рейтингом, не только
в этом документе.

Целевая квота по-прежнему не входит ни в один из двух каналов M4 (не
сопоставима по приоритету, см. §0). **Обязательное следствие для дашборда**:
рядом с M4 нужно явно показывать `target_quota_share_of_budget =
budget_places_target / budget_places_total` (из КЦП, см. §3) — не для того,
чтобы отменить решение, а чтобы пробел был виден, а не скрыт молчаливым
исключением колонки.

### M5 — Конкурс на место (только узкая бюджетная семья)
`competition_ratio = n_applications_budget_family / budget_places`.
`budget_places == 0` или программа отсутствует в КЦП → `NaN`.

### M6 — Давление серьёзного спроса (только узкая бюджетная семья)
`demand_pressure = n_serious_budget_family / budget_places`.

### M7 — Распределение приоритетов
Программа × приоритет: число заявителей, раздельно бюджетная семья /
коммерция / целевая квота (три, не две колонки — отличие от магистратуры).

### M8 — Матрицы пересечений
Программа × программа, раздельными матрицами по тем же трём каналам; ячейка
— число уникальных абитуриентов, подавшихся в обе программы этим каналом;
диагональ = M1 для этого канала (дублей внутри канала нет после схлопывания).

### M9 — Диагностика (не для основного дашборда)
- `noise_share_budget_family` — доля заявок бюджетной семьи с `priority >
  K_SERIOUS` (аналог M9 магистратуры).
- `withdrawn_share` — доля строк со статусом «Отозвано поступающим»/«Отклонено
  вузом» от общего числа строк, по каждому каналу отдельно. Новая метрика,
  которой не было в магистратуре — нужна, потому что решение по активности
  статуса (§0) принято по смыслу названия, не по документации, и должна быть
  проверяема постфактум.
- `target_quota_share_of_budget` — см. M4 выше, технически диагностика, но
  **обязательна на основном дашборде** рядом с M4 (не спрятана в
  пост-кампанию), так как это прямое предупреждение о слепой зоне M4.

## 2. M4 подробно

Формула, вывод и обоснование параметра `k` для КАЖДОГО канала по отдельности
— без изменений, см. `summer_campaign_26/docs/METRICS.md` §2 (эмпирический
Байес со сжатием, метод моментов для оценки `k`, апостериорное среднее
Beta-Binomial). Модель считается дважды, независимо: один раз на
`collapsed_budget_family`, один раз на `commercial` — у каждого канала свои
`k`/`p0` (на срезе 2026-07-08: бюджет `k≈6.46, p0≈0.271`; платное `k≈4.51,
p0≈0.272` — заметно похожий `p0`, разный `k`, платное сжимается слабее).

Объединение в `desirability_combined` (inverse-variance weighting, формула
и обоснование через корреляцию r=0.94 — см. §1) — новый слой поверх этой
модели, реализован в `compute_m4_combined()`. Референсные значения
(`k`/`p0` для обоих каналов, корреляция) — не хардкодить, брать из
`campaign_metrics_meta_bak.json` (`M4_k_budget`, `M4_p0_budget`,
`M4_k_commercial`, `M4_p0_commercial`, `M4_corr_budget_commercial`) на каждом
прогоне — эти значения дрейфуют между срезами.

## 3. Обработка краёв (обязательные правила)

- **Программы без бюджетных мест** (`budget_places == 0` или программа не
  найдена в КЦП): M4, M5, M6 — `NaN`, не 0. Вынести в отдельный файл
  `campaign_metrics_no_budget_places_bak.csv`.
- **Деление на ноль в M5/M6** — защищено, `NaN` при `budget_places == 0`.
- **Программы-«близнецы»** (10 подтверждённых случаев, одно имя и филиал,
  разный `educationProgramId`, разный `trainingDirection`) — связка везде по
  `educationProgramId`. Источник КЦП (RTF) организован по секциям
  «НАПРАВЛЕНИЕ ПОДГОТОВКИ» — сопоставлять с `raw/` нужно по (имя программы +
  `trainingDirection`/код направления) вместе, не по одному имени программы.
- **Парсинг КЦП из RTF** — по образцу `match_supply_demand.py` из
  магистратуры: `textutil -convert html`, строка программы — `<tr>`, где
  первая ячейка содержит `<a href>` (агрегатные строки-заголовки направлений
  без ссылки — отсекать). Колонки (см. `HANDOFF...md` §6.5): название,
  онлайн-признак, **Бюджетные места (КЦП)**, **Особое право**, **Целевая
  квота** (внутри встречаются сноски-ссылки вида `12 <a>1</a>` — отделять
  число от маркера сноски регэкспом), **Отдельная квота**, **Платные места**,
  **Иностранцы**. 4 файла (по филиалу) — собрать в единый
  `data/supply_demand_bak.csv` (`educationProgramId, filial, budget_places,
  budget_places_special_right, budget_places_target, budget_places_separate,
  commercial_places, foreigner_places`) до расчёта M4–M6.
- **Целевая квота: несколько пулов на одну программу** (разные заказчики) —
  агрегировать через `nunique(idEpgu)` по всем пулам вместе на этапе M1/M2,
  не суммировать `count` по пулам построчно (иначе задвоение).
- **Единственный неразрешённый автоматически человек** (`idEpgu=1785708`,
  «Международная программа по бизнесу и экономике») — правило схлопывания
  (§0) берёт минимальный priority как тай-брейк; отдельного ручного
  вмешательства при пересчёте на новых срезах не требуется, тай-брейк общий.

## 5. Как запустить

```bash
python3 scripts/parse_supply_bak.py docs/ -o data/
python3 scripts/campaign_metrics_bak.py bak_data/ data/supply_bak_raw.csv \
    --snapshot-date YYYY-MM-DD -o data/
```

Оба скрипта написаны и прогнаны на срезе 2026-07-08: 121/121 программ
сопоставлено с КЦП (0 неоднозначных близнецов), все 4 филиала сходятся с
опубликованным «Итого»/«Всего» по каждой из 6 колонок квот. Корреляция
M4×M5 на этом срезе ≈ −0.006 (ортогональность осей подтверждена сильнее, чем
в магистратуре, где было ≈ −0.15).

**Важное структурное отличие от магистратуры**, обнаруженное при написании
парсера: 4 RTF-файла НЕ единообразны по структуре (разный набор и порядок
колонок квот на разных филиалах, Пермь — без гиперссылок на программы,
собственная многоуровневая шапка). `parse_supply_bak.py` держит явную
конфигурацию колонок на каждый филиал (`FILIAL_CONFIGS`), не полагается на
одну фиксированную структуру.

**Особенность заочных/очно-заочных программ**: КЦП-таблицы озаглавлены
«ОЧНАЯ ФОРМА ОБУЧЕНИЯ» — программы с пометкой «ОЗ» (например, «Программная
инженерия (ОЗ)» в Нижнем Новгороде) в них не встречаются по объективной
причине (вне охвата таблицы, не входят в очный набор), а не из-за ошибки
сопоставления — такие программы корректно попадают в
`campaign_metrics_no_budget_places_bak.csv`.
